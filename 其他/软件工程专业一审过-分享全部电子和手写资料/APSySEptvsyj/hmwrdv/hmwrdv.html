Generally Speaking
总的来说测试技术分为黑盒测试，白盒测试和灰盒测试。
黑盒测试：
  • 边界测试
  • 特殊值测试
  • 等价类划分
  • 回归测试
  • 决策表测试
白盒测试：
  • DD-Path

黑盒测试     Black Box Testing
Tester would only know the legal inputs and what the expected outputs should be.

Boundary Value Testing     边界值测试
The most common place where error tends to appear is at the extreme values, like min, min+, nom, max- and max.


Robustness Testing          健壮性
With robustness testing, we can find what happens when the extreme are exceeded with value slightly greater then the maximum or the minimum.

Worst-case Testing     最坏情况测试
Worst-case Testing tries to find out what happens when more than one variable has an extreme value. It would generate about 5^n testing cases.

Robust Worst-case Testing     健壮性最坏情况测试
This is the most complex testing method. It detects muti-error-approaching and robustness at the same time.


Special Value Testing          特殊值测试

Random Testing          随机值测试
This is not an optimal testing tech since it has a low probability of detecting many defects. However, it does sometimes uncover defects that standardize testing techniques might not. Thus, it should be considered as an add-on testing technique.

Equivalence Partitioning     等价类划分测试
Divide input data into partitions according its type. Then generate test cases covering each partition at lease once.
There are four kinds of equivalence class testing:
  • Weak Normal Equivalence Class Testing
    • Use one variable from each equivalence class
  • Strong Normal Equivalence Class Testing
    • 覆盖所有等价类，并且每个变量都要覆盖。不用覆盖非法输入
  • Weak Robust Equivalence Class Testing
    • 覆盖所有合法输入种类，并把一个输入的非法输入全部覆盖
    • 或者这样也可以
  • Strong Robust Equivalence Class Testing
    • 覆盖全部输入样例，每个变量都要全部覆盖，每个非法输入的每个变量也都要覆盖上

Positive and Negative Testing
Since in most cases, positive case are less than negative, it follows the twenty-eighty law. 

Regression Testing     回归测试，做回归
Regression is necessary during the whole develop, maintenance and version upgrading process. It has to be implemented to avoid unintended side effects brought by changes.

Decision Table-Based Testing     基于决策表的测试
一个表格记录了每个决策变量cn，相当于一个不同决策的排列组合


White-Box Testing
White box testing is done when the test knows the inner structure of the system. How it functions, how the result is generated. So the white-box test cases are designed by examining paths or logic.

DD-Path, Decision to decision Path     决策到决策路径
省略掉过程中的顺序执行部分，将所有决策部分去除作为一个节点划出一个决策树。
对DD-Path需要使用Statement Coverage Testing

Statement Coverage Testing
测试样例覆盖掉DD-Path上的每一个分支。由于分支上各个节点之间都是顺序执行的，所以程序的每一个语句都会得到执行。

Loop Coverage Testing
对每个循环执行如下操作：
  • Skip the loop entirely
  • Only one pass through the loop
  • M passes through the loop there m<n
  • N-1, N, N+1 passes through the loop
After testing a loop, shrink it into a point.

功能测试和结构化测试
Advantage of functional testing:
  • its natural and understood by everyone. It works directly on the purpose of the system
  • Tester needs no knowledge of the implementation, nor specific programming language nor skill.
  • Tester and programmer are independent of each other.
  • Test cases can be designed as soon as the specifications are completed

Disadvantage of functional testing:
  • The input test rages is rather small.
  • Since tester has no recognition of implement, programmer can do the trick to cover the errors.

Advantage of structural testing:
  • It force developer to be very careful about implementation
  • It reveal hidden errors in code.

Disadvantage of structural testing:
  • It can’t verify whether the specifications are correct, nor it can detect missing paths and data-sensitive errors.
  • It cannot execute all possible logic path through a program because it’s far too particular
  • It’s expensive
