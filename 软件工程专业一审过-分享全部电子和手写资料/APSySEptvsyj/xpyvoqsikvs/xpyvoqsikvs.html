排序
Shell sort希尔排序/Diminishing Increment Sort

shell sort is an improvement of inserting sorting. Instead of comparing adjacent two object, Shell sort compare two object having an diminishing gap. At the start of whole process of sorting, the number sequence is very disorder, however Shell Sort takes comparable huge step, then the sequence gets more and more order, the step reduces. The running time of Shell Sort is heavily dependent on the gap sequence it uses. The Shell Sort is unstable. The gap sequence often used is Shell sequence, floor(N/2^k)(N divided by two to the power of k), where k represents the round of sorting process. The gap ends with 1.

维基百科：
Shell Sort, also known as Shell sort or Shell's method, is an in-place comparison sort. It can be seen as either a generalization of sorting by exchange (bubble sort) or sorting by insertion (insertion sort).[1] The method starts by sorting pairs of elements far apart from each other, then progressively reducing the gap between elements to be compared. Starting with far apart elements can move some out-of-place elements into position faster than a simple nearest neighbor exchange. Donald Shell published the first version of this sort in 1959.[2][3] The running time of Shellsort is heavily dependent on the gap sequence it uses. For many practical variants, determining their time complexity remains an open problem.



Bubble Sort冒泡排序
It has the computational complexity of O(n^2). In each round, like k'th round, the algorithm linear scan the sequence from the k'th number to the end, swapping the adjacent two numbers if they are in the wrong order, selecting the biggest or smallest number, float it to the first position of left sequence.

维基百科：
Bubble sort, sometimes referred to as sinking sort, is a simple sorting algorithm that repeatedly steps through the list to be sorted, compares each pair of adjacent items and swaps them if they are in the wrong order. The pass through the list is repeated until no swaps are needed, which indicates that the list is sorted. The algorithm, which is a comparison sort, is named for the way smaller elements "bubble" to the top of the list. Although the algorithm is simple, it is too slow and impractical for most problems even when compared to insertion sort.[1] It can be practical if the input is usually in sort order but may occasionally have some out-of-order elements nearly in position.



Quick Sort快速排序
Quick Sort is a commonly used divide and conquer sorting solution. It first select a number, called pivot in the field the algorithm is dealing with, then divide the field into two part, one is all less than the selected number, the other part, all object is no less than the pivot. Then the algorithm recursively solve the field until the length of the field is less or equal than one, which has no need to be ordered. The computational complexity relies on the pivot selection. In the worst case, the algorithm can fall back to Omega N square(O(n^2)). So we can select the pivot randomly. So from the probability theory, it’s N*logN.


维基百科：
Quicksort is a divide and conquer algorithm. Quicksort first divides a large array into two smaller sub-arrays: the low elements and the high elements. Quicksort can then recursively sort the sub-arrays.

The steps are:

    1. Pick an element, called a pivot, from the array.
    2. Reorder the array so that all elements with values less than the pivot come before the pivot, while all elements with values greater than the pivot come after it (equal values can go either way). After this partitioning, the pivot is in its final position. This is called the partition operation.
    3. Recursively apply the above steps to the sub-array of elements with smaller values and separately to the sub-array of elements with greater values.

The base case of the recursion is arrays of size zero or one, which never need to be sorted. A quicksort that sorts elementslo through hi (inclusive) of an array A can be expressed as function quicksort(A, lo, hi). Sorting the entire array is accomplished by calling quicksort(A, 1, length(A)).

Heap Sort堆排序
Heap sort is a compatible sorting strategy of quick sort. Heap Sort sort the number array by establishing then maintaining a heap in the number array. In each round the algorithm pick then remove the top of the heap, replacing it with another number of the heap, usually the number at the bottom of the heap, then maintain the heap to its normal shape. After N rounds, the array become sorted. Each round, the maintaining job has the computational complexity of logN.

Branch and Bound Algorithm 分支限界法
The algorithm explores branches of this tree, which represent subsets of the solution set. Before enumerating the candidate solutions of a branch, the branch is checked against upper and lower estimated bounds on the optimal solution, and is discarded if it cannot produce a better solution than the best one found so far by the algorithm.
搜索过程中遇到可能的解并将这个解作为问题的下界。将不可能得到更优解的状态空间剪去。

A*算法 A* Algorithm
在过程中的每一步进行估算g(x)和h(x)，g(x)表示当前代价，h(x)表示估算剩余代价。h*(x)表示理想估计剩余代价。要保证估算规则：使得 h(x) <= h*(x)。则当前决策造成的估计代价是 f(x)=g(x)+h(x). 使用Best First思想来进行拓展下一个节点。需要证明一个定理：
          使用Best-first策略搜索树, 如果A*选择的节点是目标节点, 则该节点表示的解是优化解。

As an example, when solving Single-Source Shortest-Path Problem, we can use A Star Algorithm. We can let h(x)=min({e|e is the outgoing edge of x}). In each round we pick a vertex according to Best First rule and loose all nodes incident from the chosen vertex. When the destination node is chosen, we get the shortest path length.
